# Russian i18n Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add full Russian localization support by providing a complete `ru` translation file and registering it in the frontend i18n setup.

**Architecture:** The frontend already loads language resources from `src/ui/locales/*.json` through `src/ui/i18n.ts`. This change keeps that pattern intact: add one new locale file that mirrors the existing English key structure exactly, then register it in the i18n resources map so the language detector can select Russian at runtime. The rest of the app should consume the new strings automatically through the existing `react-i18next` wiring.

**Tech Stack:** React, TypeScript, i18next, react-i18next, i18next-browser-languagedetector


Expected: FAIL with `ru.json is missing`.

**Step 2: Write the minimal implementation**

Create `src/ui/locales/ru.json` and mirror the full key tree from `src/ui/locales/en.json`.

Implementation rules:
- Keep every key path identical to English.
- Preserve interpolation placeholders exactly, including names like `{{count}}`, `{{name}}`, and `{{provider}}`.
- Preserve plural forms such as `_one` / `_other`.
- Translate all visible UI text into natural Russian.
- Keep product names, protocol names, and code-like tokens unchanged where appropriate, such as `MCP`, `TTS`, `STT`, `Live2D`, `Ollama`, `OpenAI`, `whisper.cpp`, and HTML snippets inside strings.

**Step 3: Run the verification check again**

```bash

```

Expected: PASS.

**Step 4: Sanity-check the locale shape**

```bash
node -e "const en=require('./src/ui/locales/en.json'); const ru=require('./src/ui/locales/ru.json'); const diff=(a,b,p='')=>{for(const k of Object.keys(a)){if(!(k in b)) throw new Error('Missing key: '+p+k); if(a[k]&&typeof a[k]==='object'&&!Array.isArray(a[k])) diff(a[k],b[k],p+k+'.');} for(const k of Object.keys(b)){if(!(k in a)) throw new Error('Extra key: '+p+k);}}; diff(en,ru); console.log('Key parity OK');"
```

Expected: `Key parity OK`.
